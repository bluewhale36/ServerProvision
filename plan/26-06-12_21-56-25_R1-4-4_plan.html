§0. 동기 — "예외 중의 예외" 사후 패치의 구조 정리

직전 커밋 8ab2bf2 는 ISO 등록의 FS-then-DB 비원자성 결함(파일은 디스크에 써졌으나 후속 DB insert/update·마커 발급 실패 시 트랜잭션이 파일을 되돌리지 못해 오펀 발생)을 오펀 복구 saga 로 메웠다. 이 상황은 Notion 설계 단계에서 고려조차 못 했던 진짜 예외였기에 사후 패치 성격이며, 다음 두 부채를 남겼다.

  • Runner 의 multi-catch — 실패 종류가 늘 때마다 catch 줄이 자라는 분기문. CLAUDE.md §조건분기 무분별 확장 금지(불가침)와 정면 긴장. 누락 시 silent 오분류 사고(아래 §1).
  • @Lazy 순환 — Launcher→Runner→OrphanIsoRecoveryService→Launcher 생성자 순환을 @Lazy 로 임시 차단. 프로젝트의 4번째 @Lazy 누적 — 구조 신호.

또한 같은 FS-then-DB 위험은 BIOS · BMC · Subprogram 에도 동일하게 존재하나, 이들은 cleanupFailedUpload 로 무조건 파괴적 정리(일시 장애에도 업로드 트리 유실)한다 → ISO 만 보호받는 비대칭. R1-4-4 는 이 패치를 예외 다형성 + global SPI 로 끌어올려 도메인 횡단으로 적용 가능한 seam 으로 정리한다(행동 변화 0).

R 시리즈 정합 — R1-4-1~3 이 ISO 의 lifecycle/registration/integrity 책임을 service 로 분리한 연장선. 본 슬라이스는 그 위에 쌓인 사후 패치를 같은 "분리 + 다형성" 방식으로 정돈한다.

§2. 요구사항 (확정) + 비목표
  1. 실패 처분의 다형성 — 등록 실패 예외가 RegistrationFailure.disposition() 으로 자기 처분(CLEANUP / PRESERVE / QUARANTINE)을 선언. Runner 의 분류 catch 체인 소멸 → catch(RegistrationFailure) 단일 + 프레임워크 경계 어댑터 1곳.
  2. sealed exhaustiveness — 새 등록 실패 예외 추가 시 처분 선언을 컴파일 타임에 강제(누락 불가). §1 결손1 사고 구조적 차단.
  3. 오펀 saga 의 global 승격 — OrphanIsoQuarantine → global/orphan/OrphanQuarantine(resource_type 키). 격리/복구 본체를 도메인 무관 인프라로. 도메인은 OrphanRecoverySpi 구현체로 "재등록 방법" 만 공급.
  4. @Lazy 제거 — record/purge(OrphanQuarantineService, launcher 무관) ↔ retry/discard(OrphanRecoveryService→SPI→Launcher) 분리로 의존 그래프 DAG화.
  5. 행동 보존(불가침) — ISO 의 기존 동작·URL·복구 UX·typed-name 가드·TTL reaper 전부 동일. 기존 OrphanIsoRecoveryControllerTest 가 회귀망.
  6. retry 멱등 보강 — retry() 의 relocate 가 "이미 active 에 있으면 skip" 이도록(복구 경로 안의 FS-DB 비원자성 재귀 방어, §8).

     | Duplicate

     | DuplicateFilename e)    { fail }     // ← CONTENT

AFTER — disposition() 다형성

try { ... }
catch (RegistrationFailure f) {   // 예외가 자기 처분 보유
    switch (f.disposition()) {
      case Cleanup c   -> failOnly(...)
      case Preserve p  -> failPreserve(...)
      case Quarantine q-> quarantine(q.failureClass())
    }                              // sealed → exhaustive
}
catch (DataIntegrityViolationException e) {   // Spring 경계 어댑터 (유일 잔존)
    quarantine(DB_CONSTRAINT);
}

§4. 도메인 모델 (SSOT)

① 처분 계약 (global/registration) — 다형성 핵심

// 등록 실패 예외가 구현. 처분을 "정의 지점" 에서 선언.
public interface RegistrationFailure {
    FailureDisposition disposition();
}

// 처분 = sealed (Quarantine 은 OrphanFailureClass 동반 데이터)
public sealed interface FailureDisposition
        permits FailureDisposition.Cleanup,
                FailureDisposition.Preserve,
                FailureDisposition.Quarantine {
    record Cleanup()   implements FailureDisposition {}          // finalize 가 파일 정리 — 격리 X
    record Preserve()  implements FailureDisposition {}          // nudge — 임시 보존
    record Quarantine(OrphanFailureClass failureClass) implements FailureDisposition {}  // 인프라/일시 — 격리
    Cleanup  CLEANUP  = new Cleanup();
    Preserve PRESERVE = new Preserve();
}
