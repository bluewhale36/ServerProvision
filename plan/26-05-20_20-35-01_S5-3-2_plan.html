1. 현재 상태 — disabled button 의 tooltip 침묵

1.1 마크업 현황

S5-2-4 가 신설한 nudge modal fragment 의 ReplaceBtn :

<button th:id="${prefix} + 'ReplaceBtn'"
        class="n-btn n-btn-danger"
        disabled
        title="대체할 기존 자원을 목록에서 선택 후 자원명을 정확히 입력하세요">
    기존 영구 삭제 후 등록
</button>

1.2 브라우저 native 의 한계

HTML 표준 동작 :

  • <button disabled> 는 pointer 이벤트를 받지 않음 → title 의 hover tooltip 이 표시되지 않음
  • 또한 cursor: not-allowed 도 disabled 상태에서는 일부 브라우저가 적용 안 함

1.3 사용자 영향

  • 사용자가 nudge modal 진입 직후 — 자원을 아직 선택하지 않은 상태 → ReplaceBtn 회색 비활성
  • 사용자가 "왜 안 눌리지?" 의문 → 위에 hover 해도 안내 없음 → 혼란
  • 이전 작업 시 사용자 본인 메모 : "그래.. 뭐 툴팁은 disable 일 때 동작 안하지만..하"
2. 요구사항
  1. ReplaceBtn 이 비활성 상태일 때도 hover 시 안내 메시지가 표시되어야 함
  2. 버튼 자체의 click 동작은 비활성 상태에서 그대로 차단되어야 함 (data 안전)
  3. 활성화된 상태에서는 tooltip 노출 안 함 (정상 hover 시 시각 노이즈 회피)
  4. 마우스 / 키보드 양쪽에서 접근 가능 (a11y) — focus 시도 tooltip 보이게
  5. 구현은 fragment 1 곳 + 작은 CSS 추가만 — 6 도메인 자동 흡수
🎬 패턴 비교 — disabled 버튼 위에 마우스 올려보세요

3 가지 후보 패턴을 라이브로 비교 — 각 카드의 빨간 버튼 위에 hover 해보세요.

A 현재 (native)
tooltip 안 뜸 (브라우저가 disabled 버튼의 title 표시 안 함)
B wrapper title
wrapper span 이 title 보유 — 일부 브라우저에서만 표시 + tooltip 위치 어색
C ✓ CSS pseudo tooltip
wrapper 의 ::after 가 자체 tooltip 렌더 — 모든 브라우저 작동 + 디자인 통일 + arrow 추가 가능

3 패턴 비교표

패턴모든 브라우저 작동디자인 일관성a11y구현 부담비고
A — 현재 ❌ tooltip 미표시 — ❌ 0 본 슬라이스가 해결할 대상
B — wrapper title ⚠ Safari 표시, Chrome 일관성 없음 OS native tooltip — 페이지와 비통일 ⚠ 의미 부정확 마크업만 구현은 가벼우나 OS tooltip 모양 통제 불가
C — CSS pseudo tooltip ✅ ✅ 페이지 디자인 토큰 일관 ✅ focus-within 추가 가능 fragment + CSS ~30 line 채택안
3. 채택안 — CSS pseudo tooltip + wrapper span

3.1 마크업 변경 (fragment)

<span class="n-btn-tooltip-wrap"
      th:data-tooltip="...메시지...">
    <button th:id="${prefix} + 'ReplaceBtn'"
            class="n-btn n-btn-danger"
            disabled>
        기존 영구 삭제 후 등록
    </button>
</span>

핵심 :

  • wrapper span 이 data-tooltip 속성 보유 — 모든 hover/focus 이벤트의 진입점
  • button 의 title 속성 제거 — OS native tooltip 중복 표시 회피
  • JS 가 button 활성화 / 비활성화 시 wrapper 의 data-tooltip-active 토글 — 활성화된 상태에선 tooltip 숨김

3.2 CSS 추가 (1 곳 — global/style.css 또는 confirm-modal.css 의 일부)

/* S5-3-2 — disabled 버튼에 hover tooltip 표시. wrapper 의 ::after 활용. */
.n-btn-tooltip-wrap { position: relative; display: inline-block; }

.n-btn-tooltip-wrap[data-tooltip]:not([data-tooltip-active="false"]):hover::after,
.n-btn-tooltip-wrap[data-tooltip]:not([data-tooltip-active="false"]):focus-within::after {
    content: attr(data-tooltip);
    position: absolute;
    bottom: calc(100% + 6px);
    left: 50%;
    transform: translateX(-50%);
    background: rgba(31, 35, 40, 0.92);
    color: #fff;
    padding: 6px 10px;
    border-radius: 4px;
    font-size: 12px;
    white-space: nowrap;
    pointer-events: none;
    z-index: 1100;
}

.n-btn-tooltip-wrap[data-tooltip]:not([data-tooltip-active="false"]):hover::before,
.n-btn-tooltip-wrap[data-tooltip]:not([data-tooltip-active="false"]):focus-within::before {
    content: '';
    position: absolute;
    bottom: calc(100% + 1px);
    left: 50%;
    transform: translateX(-50%);
    border: 5px solid transparent;
    border-top-color: rgba(31, 35, 40, 0.92);
}

3.3 JS 변경 (nudge-modal.js)

conflict 자원 선택 시점에 wrapper 의 data-tooltip-active="false" 토글로 tooltip 숨김 :

// 자원 선택 시 (li.addEventListener('click', ...))
//   기존 :
//     replaceBtn.disabled = false;
//     replaceBtn.removeAttribute('title');
//   변경 :
const wrap = replaceBtn.closest('.n-btn-tooltip-wrap');
li.addEventListener('click', () => {
    ...
    replaceBtn.disabled = false;
    if (wrap) wrap.dataset.tooltipActive = 'false';  // tooltip 비활성
});

// 초기화 시점 (modal open) :
replaceBtn.disabled = true;
if (wrap) delete wrap.dataset.tooltipActive;  // tooltip 다시 활성 (default)
장점 마크업 1 fragment + CSS ~25 line + JS 2 줄 — 6 도메인 자동 흡수. 디자인은 페이지 토큰 일관. focus-within 으로 a11y 확보 (Tab key 로 진입해도 tooltip 노출). 활성 후 tooltip 안 보이는 정합성 유지.

6. 테스트 — DOM 동작 검증

단위 테스트 한계

CSS pseudo-element 의 hover 효과는 JSDOM / MockMvc 로 검증 불가능 — 브라우저 렌더링 의존. JS 핸들러 동작 만 단위 테스트 + 브라우저 E2E 가 시각 확인.

가능한 자동 테스트 (3 시나리오)

  1. (20) nudge open 시점 — wrapper 의 data-tooltip-active 부재 (= tooltip 활성)
  2. (21) 자원 선택 시점 — wrapper 의 data-tooltip-active=false (= tooltip 비활성)
  3. (22) modal 닫고 다시 open — data-tooltip-active 다시 부재 (초기화)

이는 JS 핸들러의 DOM 변형 검증 — Jest / vitest 없는 프로젝트라 가능하면 jsdom 환경에서 spring controller test 와 분리. 또는 manual 검증 표 작성.
