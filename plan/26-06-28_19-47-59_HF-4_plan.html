§1. 현재 상태 · 목표

브라우저 테스트(2026-06-28)에서 발견된 업로드 폼 UX 2건. 둘 다 R2-3(백엔드 예외 라우팅)과 무관한 frontend/라벨 이슈로, R2-3 commit 후 본 hotfix 로 처리.

① 검증 메시지 잔존 (UX 버그)

파일 0개로 "번들 등록" 클릭 → showError('업로드할 파일이 없습니다.')(uploadError 박스). 이후 파일을 선택해 조건이 충족돼도 에러 메시지가 안 사라진다. 원인: resetError() 가 startXhrUpload() 안(성공 경로)에서만 호출되고, 파일 input 변경을 듣는 클리어 핸들러가 없다. submit 시작부의 FormError.clear(form) 은 필드 에러만 지우지 uploadError 박스는 안 건드림. bmc-new.js:109 / bios-new.js:130 / subprogram-new.js:143 동일 복붙.

② 디렉토리 생성 라벨 (의도 확정, option 1)

DefaultTargetDirectoryPolicyService 는 allowCreateDirectory 로 target 의 '부모' 디렉토리 생성만 게이팅(.getParent() 검사). 말단 target 은 번들 추출이 항상 생성(부모 존재 시). 미체크라도 1-depth 는 생성, 2-depth(부모 부재)부터 거절 — 부모-게이팅이 의도된 설계로 확정(정책 코드 무변경). 단 라벨 "디렉토리 생성 허용 / 경로가 존재하지 않을 경우 디렉토리를 생성합니다"가 "미체크=무생성"으로 오해를 부르므로 라벨/힌트만 정정.

목표 — ① 파일 선택 즉시 "없습니다" 메시지가 사라지게(공통 1곳 수정으로 3 도메인 동시) ② 라벨/힌트를 부모-게이팅 실제 동작에 맞게 정정. frontend/template 만 — 백엔드·정책·예외 라우팅 무변경.
§2. 설계 (DRY 공통화 / 라벨 정정)

① — 공통 배선(bundle-upload-bootstrap.js)

bindModeTabs 가 이미 파일 input 3종(folder/zip/single)을 다룬다(mode 전환 시 clearIfPresent). 여기에 파일 input change → 콜백 배선을 추가하고, 각 도메인이 onFilesChange: resetError 를 넘긴다. 3곳 복붙 대신 공통 1곳 + 도메인당 콜백 1줄 — CLAUDE.md §중복 금지 정합.

// bundle-upload-bootstrap.js  bindModeTabs(opts) 내부 — 신규 배선
[folderInput, zipInput, singleInput].forEach(inp => {
    if (inp) inp.addEventListener('change', () => {
        if (typeof opts.onFilesChange === 'function') opts.onFilesChange();
    });
});

// 각 도메인 new.js — bootstrap.bindModeTabs({ ..., onFilesChange: resetError })
// + submit 시작부에 resetError() 1줄(재제출 stale 방지)

② — 라벨/힌트 정정(template)

현재 (오해 유발)
디렉토리 생성 허용
경로가 존재하지 않을 경우 디렉토리를 생성합니다.
정정안 (실제 동작)
상위 디렉토리 생성 허용
대상 경로의 상위(부모) 디렉토리가 없을 때 함께 생성합니다. 대상 디렉토리 자체는 업로드 시 생성됩니다.

문구는 예시 — D2 에서 확정. 정책 로직은 건드리지 않는다.

🎬 라이브 — 파일 선택 시 메시지 클리어
동작 ① "등록"(파일 0개) → 에러 → ② "파일 선택"
파일 0개
업로드할 파일이 없습니다.

수정 전: 파일을 선택해도 에러가 그대로. 수정 후: 파일 선택 즉시 에러가 사라짐(파일 input change → resetError).
