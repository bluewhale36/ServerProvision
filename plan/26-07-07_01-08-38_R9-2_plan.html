§2. 요구사항 · 설계 결정 (D1~D5, 대안 비교)

D1. DriftKind 에 순수 데이터 4필드 — label · description · recommendedAction · autoApplicable 채택

생성자 필드 + 접근자(IntegrityStatus.displayMessage 선례와 동일 위치·동일 방식). 12종 전부 부여 — 미생성 6종(RESOURCE_RENAMED 계열 등)도 javadoc 의미가 이미 확정돼 있고, 상수별 유무 분기를 만들지 않는 쪽이 분기 금지 원칙에 부합. 각 상수 javadoc 에 "탐지 로직 미구현(MK3-2 소관)" 사실 명기.
비채택: ⓐ method-per-constant — 행위가 아닌 순수 데이터라 필드가 더 단순. ⓑ MessageSource i18n — 단일 언어 운영, enum 필드가 SSOT 로 더 단순(조사 보고 §7-7). ⓒ 라벨만 부여 — "무슨 일인지+뭘 하면 되는지"가 불만 ④ 의 본체라 3필드가 최소.

D2. 소비 방식 — 템플릿이 enum 접근자를 직접 읽음, DriftResponse 무변경 채택

DriftResponse.kind 가 이미 DriftKind 타입이므로 Thymeleaf 가 ${drift.kind.label}/${drift.kind.autoApplicable} 로 접근자를 직접 호출한다. DTO 필드 4개 복제가 사라지고 enum 이 유일 SSOT. REST JSON 은 기존대로 enum 이름 직렬화(외부 API 계약 무변경 — ReconciliationRestControllerTest 의 $.drifts[0].kind 단언 보존).
비채택: DriftResponse 에 label 등 4필드 추가 — 같은 값을 두 층에 복제(중복 금지 위반)하고 REST 소비자가 없는데 페이로드만 증가.

D3. autoApplicable SSOT 3자 공유 — 서버 가드 · 버튼 노출 · 배지 색이 같은 접근자 호출 채택

PathReconciliationService 의 private isAutoApplicable(kind) 5종 나열을 삭제하고 가드가 drift.getKind().isAutoApplicable() 호출. 템플릿 3곳도 ${drift.kind.autoApplicable} — 하드코딩 4종 나열 ×3 제거. GHOST_DB_ROW 버튼이 자연 노출된다(Q5 확정) — hard-delete 의미이나 대상 파일이 애초에 없어 안전한 정리이고, 같은 액션이 휴지통 화면에 이미 노출돼 있어(TrashController) 오히려 現비노출이 비일관. 문구는 D1 의 recommendedAction("복구할 파일이 없어 안전한 정리")이 안내하고, confirm modal 의 어휘 정합은 R9-3 이 담당.

D4. 전역 auto-apply OFF → UI 사전 차단, 서버 가드와 단일 메서드 공유 채택

PathReconciliationService 에 public isAutoApplyEnabled() 신설(!Boolean.FALSE.equals(autoApplyEnabled) — null=활성 유지). 서버 가드(apply)와 컨트롤러 뷰모델(autoApplyEnabled 플래그)이 이 한 메서드를 함께 호출 — childEnableBlockReason() 선례의 SSOT 패턴. 템플릿: OFF 면 [자동 적용] 버튼 disabled + title tooltip("전역 자동 적용이 꺼져 있어요 (reconciliation.auto-apply). 켠 뒤 다시 시도하세요."). 서버 가드는 direct POST 안전망으로 유지(발동 시 문구는 D5).
주의(비판 검증 리스크 4): 참조 플래그는 반드시 reconciliation.auto-apply(수동 버튼 가드와 동일) — 스캔 중 무인 자동 적용용 auto-apply-path-drift/auto-apply-ghost-row 와 혼동 금지.

D5. 예외 메시지 발생 지점별 정확화 — static factory 2종 채택

DriftAutoApplyNotAllowedException.notApplicable(kind)("'{label}' 종류는 시스템이 자동 정합할 수 없습니다…") / .globalOff()("전역 자동 적용 옵션이 꺼져 있어…"). 보조: toResponse 의 스캔 소요를 사람 문구로 포맷("0.45초"/"1분 3초"), 빈 상태 프로퍼티 키 제거. flash 문구는 R9-3 에서 응답 체계가 바뀌므로 본 슬라이스에서 건드리지 않는다.
