§2. 슬라이스 요구사항

본 슬라이스는 ISO 등록 도메인 응집. R1-4-1 의 IsoLifecycleService 와 동형 패턴이지만 LifecycleService interface 의 두 번째 추가 구현체는 아니다 — 등록은 lifecycle 의 두 진입점 (toggle / softDelete 등) 과 시그니처가 다르고, 4 도메인 (OS/BIOS/BMC/Subprogram) 중 ISO 만의 고유 흐름이라 interface 추상화 대상이 아님.

  1. 책임 응집 — ISO 등록 5 public + 7 private helper + 1 record (PreparedIsoRegistration) 를 IsoRegistrationService 로 응집
  2. 의존 비대칭 해소 — OSMetadataService 가 자식 ISO 의 등록 / nudge 정리 책임을 떠안던 비대칭을 단방향 (등록 service → lifecycle service / marker / repo) 정렬
  3. OSNudgeService 의 호출처 갱신 — 4 호출 (proceed.completePending, replace.purgeIsoForNudge + completePending, replaceIntent.purgeIsoForNudge) 의 lookup target 을 새 service 로
  4. IsoUploadController 의 호출처 갱신 — 2 호출 (prepareIsoRegistration overload 2종) lookup target 갱신
  5. IsoRegistrationRunner 의 호출처 갱신 — 1 호출 (finalizePreparedIsoRegistration)
  6. 외부 endpoint 동일 — Controller URL / HTTP method / response / Job event 100% 동일
  7. 의존성 strategy 보존 — postCreationTaskStrategies (S5-7) 의 List<PostCreationTaskStrategy> 주입 자리도 함께 이동. trigger 책임이 등록 service 로 응집
비 목표 (out of scope) :
  • ISO 무결성 검증 (verifyIntegrity / verifyAndRecordIntegrity) → R1-4-3 (IsoIntegrityService)
  • ISO 의 단순 조회 (findISO / findIntegrityStatus) — OSMetadata 응답 조립의 일부라 OSMetadataService 잔류
  • ISO 의 updateISO (편집) — 등록이 아닌 후속 수정. 잔류 (또는 R1-4-3 으로 함께 이동 검토)
  • BIOS / BMC / Subprogram 의 동형 패턴 환원 → R2
  • OSMetadata 생성 흐름 (create / completePendingOSMetadataFromNudge / purgeOSMetadataForNudge) — 부모 도메인이라 잔류
스코프 한 줄 정의 — "OSMetadataService 가 ISO 의 생애 가운데 '태어나는 순간' 만 떠안고 있는 비대칭을 IsoRegistrationService 로 풀어내는 슬라이스". 이로써 OSMetadataService 는 OSMetadata (parent) 의 CRUD + 조회 응답 조립만 담당, ISO 의 등록 / 정리 / lifecycle 은 각자의 service 가 응집 (R1-4-2 후 R1-4-3 으로 완결).

3.2 메서드명 정렬 — service 컨텍스트 노출 어휘로 단축

OSMetadataService 의 메서드명은 "이 service 가 OSMetadata 의 것이라 ISO 명시 필요" 였지만, 신 service 안에선 이미 도메인 컨텍스트가 ISO 등록이므로 단축 :

// 옛 (OSMetadataService 안)                          // 새 (IsoRegistrationService 안)
osMetadataService.addISO(osId, req, file)             →  isoRegistrationService.add(osId, req, file)
osMetadataService.prepareIsoRegistration(...)         →  isoRegistrationService.prepare(...)
osMetadataService.finalizePreparedIsoRegistration(...)→  isoRegistrationService.finalize(...)
osMetadataService.completePendingIsoFromNudge(...)    →  isoRegistrationService.completePendingFromNudge(...)
osMetadataService.purgeIsoForNudge(...)               →  isoRegistrationService.purgeForNudge(...)

이 정렬은 R1-4-1 에서 toggleIsoEnabled → toggleEnabled, softDeleteISO → softDelete 같은 단축과 동일 정책.
