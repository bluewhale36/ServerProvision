7. 예외 / 회귀

예외 신설

없음 — UI 정합화 슬라이스.

회귀 위험

  • BIOS / BMC 의 /new 매핑 — 기존 /{boardId}/new path 와 충돌 가능성. Spring 의 path matching 은 /new 가 /{boardId}/new 보다 specific 하므로 우선. 안전.
  • 등록 폼의 board 선택 dropdown — 기존 path variable 에서 받던 boardId 를 form 의 별도 필드로 받는 분기 추가. 기존 /{boardId}/new 진입로는 board 가 fixed (hidden field 로 prefill), 새 /new 진입로는 dropdown 활성. 두 분기 모두 polymorphic 으로 통합 (Spring @PathVariable(required=false) 활용).
  • os-list.js 의 emptyState 전환 — 기존 EMPTY_VERSION_MSG 상수가 OS 의 특정 텍스트만 알고 있음. 도메인별로 다른 텍스트를 처리해야 하므로 data 속성 기반으로 전환. 다른 도메인 (Board / BIOS / BMC) 도 같은 JS 를 공유하므로 일괄 적용.
  • 휴지통 이모지 제거 — 이모지가 button label / option 텍스트 / details summary 에 사용. 제거 후에도 의미 명확하도록 텍스트 다시 검토 ('직접 입력' / '고급 설정' 등은 이모지 없이도 자명).
주의 — 외부 + 등록 버튼 신설의 추가 부담 BIOS / BMC 에서 board 선택 dropdown 추가는 폼 검증 분기 + Service 호출 시점 boardId 결정 책임 분산을 유발. 기존 BiosService.create(boardId, request) 시그니처는 그대로 두고 컨트롤러가 path 또는 form 양쪽에서 boardId 얻어 service 호출 — controller 분기 1 개 추가. 큰 영향 없음.
