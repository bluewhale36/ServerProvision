1. 현재 상태

1.1 마커 파일 발급 현황

  • ProvisionMarkerService.write(Path, MarkerLayout, MarkerContent) 가 모든 자원의 마커 파일 진입점
  • 발급 대상 :
    • BIOS 번들 (IN_TREE) — <treeRoot>/.provision.json
    • OS ISO (SIDECAR) — <isoPath>.provision.json
    • BMC firmware (SIDECAR)
    • Subprogram bundle (IN_TREE) — Driver / Utility
  • 현재 Files.write(target, bytes) 호출 후 OS-level 속성 부여 없음. . prefix 만으로 Unix-style hidden 처리

1.2 현재 가시성 매트릭스

Linux ✓

  • dot-prefix → ls 기본 숨김
  • ls -a 시 표시
  • S5-9 의 path-browser 에서 Files.isHidden() true
  • 운영 환경 — 별도 작업 불필요

macOS △

  • dot-prefix → Finder / ls 숨김
  • Cmd-Shift-. 시 표시
  • chflags hidden 추가 가능 (UF_HIDDEN) — Java 표준 API 없음, JNA 필요
  • 본 슬라이스 : dot-prefix 그대로 활용 (UF_HIDDEN 미적용)

Windows ✗

  • dot-prefix → 자동 숨김 처리 안 됨
  • 탐색기 / dir 명령에서 노출 — 사용자 혼란 가능성
  • attrib +h 또는 Java 의 Files.setAttribute("dos:hidden", true) 필요
  • 본 슬라이스의 핵심 타겟
관찰 본 프로젝트 운영 환경 (PXE 부팅 서버) 는 Linux 가 표준. 그러나 개발 / 검증 / 마이그레이션 시 Windows + Java 환경에서 자원 마커 파일이 노출되는 경우 있음. S5-10 은 cross-platform 안전망으로 dot-prefix + DOS hidden 둘 다 부여.
2. 요구사항 (확정)

2.1 Notion 명세 (S5-10 본문)

모든 *.provision.json 파일을 숨김 처리하여 기본적으로 쉽게 탐색되지 않도록 하는 작업.

2.2 채택 정책

  • 핵심 : 모든 .provision.json 마커 파일에 OS-level hidden 속성 부여
  • 구현 : Files.setAttribute(path, "dos:hidden", true) — Windows 적용, Linux/macOS UnsupportedOperationException 발생 → fail-safe try-catch 흡수
  • 호출 위치 : ProvisionMarkerService.write() 의 Files.write(target, bytes) 직후
  • Linux/macOS : dot-prefix 자체로 hidden — DOS attribute 설정 실패해도 가시성 동일하게 hidden 유지
  • 읽기 호환성 : DOS hidden 속성이 있는 파일도 Files.readAllBytes 로 정상 읽힘 — 마커 read / verify 영향 없음

2.3 추가 제약 (불가침)

  • MarkerContent 의 JSON 구조 그대로
  • HMAC 서명 / manifestHash 계산 로직 그대로
  • 마커 file name (.provision.json) 그대로 — dot-prefix 보존
  • 기존 발급된 마커 파일은 본 슬라이스가 자동 migrate 안 함 — 재발급 (재서명) 시점에 hidden 부여 (별 작업 = 마커 secret 회전)
  • S5-9 의 BE hidden 판별 (name.startsWith('.') || Files.isHidden(p)) 가 모든 케이스 cover — 본 슬라이스의 효과는 Windows 환경에서 가시

2.4 잠재적 risk : Windows 에서 read 권한

DOS hidden 속성이 있는 파일도 일반 read API 로 읽힘. ACL / permission 변경 없음. 단지 탐색기 default 뷰에서 안 보일 뿐. Files.exists, Files.readAllBytes 등 표준 API 정상 동작. 회귀 위험 0.

// ProvisionMarkerService.write() — S5-10 전
public void write(Path resourcePath, MarkerLayout layout, MarkerContent content) {
    Path target = resolveMarkerFile(resourcePath, layout);
    byte[] bytes = om.writeValueAsBytes(content);
    if (layout == MarkerLayout.IN_TREE) Files.createDirectories(resourcePath);
    Files.write(target, bytes);
    // ← 끝
}

// ProvisionMarkerService.write() — S5-10 후
public void write(Path resourcePath, MarkerLayout layout, MarkerContent content) {
    Path target = resolveMarkerFile(resourcePath, layout);
    byte[] bytes = om.writeValueAsBytes(content);
    if (layout == MarkerLayout.IN_TREE) Files.createDirectories(resourcePath);
    Files.write(target, bytes);
    applyOsHiddenAttribute(target);  // ← S5-10 신규
}

private void applyOsHiddenAttribute(Path p) {
    try {
        Files.setAttribute(p, "dos:hidden", true);
    } catch (UnsupportedOperationException | IOException ignored) {
        // Linux/macOS 에서는 DOS attribute view 미지원 — dot-prefix 자체가 hidden 관례.
    }
}

6. Step 8 테스트 시나리오

6.1 신규 시나리오 — 1 개

  • Hidden attribute fail-safe — Linux 테스트 환경에서 Files.setAttribute(p, "dos:hidden", true) 가 UnsupportedOperationException 발생해도 marker write 가 정상 성공하며 후속 read() + verifySignature() 가 정상 동작

6.2 회귀 대상

  • ProvisionMarkerServiceTest 전체 — write / read / signature / hash 모두 영향 없음 확인
  • BiosService / OSImageService / BmcService / SubprogramService 의 upload / addIso 단위 테스트 — 마커 발급 후 검증 정상
  • 전체 ./gradlew test
7. 예외 / 회귀

7.1 신규 예외 — 없음

setAttribute 가 던지는 모든 예외 (UnsupportedOperationException / IOException) 는 marker write 의 결과에 영향 주지 않도록 흡수. 마커 자체가 정상 기록됐다면 hidden flag 부여 실패는 부수 효과만 누락.

8. 부산물 · 주의

8.1 스코프 경계

  • 기존 발급된 마커 파일에 대한 일괄 hidden 부여 — 본 슬라이스 외 (마이그레이션 도구는 별 슬라이스)
  • macOS UF_HIDDEN flag (JNA 필요) — 본 슬라이스 외. dot-prefix 로 충분
  • Linux 의 별도 hidden 메커니즘 (extended attribute) — 본 슬라이스 외

8.2 S5-9 와의 시너지

S5-9 의 BE 가 name.startsWith('.') || Files.isHidden(p) 로 hidden 판별. 본 슬라이스 적용 후 :

  • Linux : dot-prefix 만으로 S5-9 가 회색 표시
  • Windows : DOS hidden 속성으로 S5-9 가 회색 표시 (Linux 에서는 부여 안 됐어도 dot-prefix 가 우선)
  • macOS : dot-prefix 로 S5-9 회색 표시
