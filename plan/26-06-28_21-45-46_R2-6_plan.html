§1. 현재 상태 · 목표

R2 묶음의 마지막 잔여. frontend 에 흩어진 alert() 에러 표시를 공통 ErrorModal 로 수렴하고, 재검증에서 R2-5 를 축소·흡수(nudge 만료 판정을 메시지 문자열 → 안정 머신 code 로). R2-4(ProblemDetail)는 drop.

현 상태 (실측)

  • error-modal.js — show({title,message,status}) / hide() 만 export. fromResponse 없음 → 호출부가 직접 await resp.json(); ErrorModal.show(...) 하거나 alert().
  • alert() 가 7 파일에 잔존 — 일부는 진짜 에러 표시(modal 화 대상), 일부는 confirm/결과알림/fallback(비대상).
  • nudge-modal.js:89 — body.message.indexOf('만료') >= 0 로 만료 판정(fragile string). NudgeSessionExpiredException 은 409 지만 응답에 code 없음.
목표 — ① ErrorModal.fromResponse(resp,opts) 로 "응답 바디 파싱(json detail||message / text) → modal 표시"를 1곳에 응집. ② 진짜 에러 alert() 만 ErrorModal 로 라우팅(confirm·결과알림·delete-reject 사전체크는 보존). ③ R2-5: NudgeSessionExpired 에 code="NUDGE_SESSION_EXPIRED" 부여 + 프론트는 message 대신 code 로 판정.

🎬 라이브 — alert vs ErrorModal.fromResponse
방식 응답

현재(alert)는 브라우저 기본 경고창 + 바디 파싱이 호출부마다 제각각. R2-6 은 fromResponse 가 content-type 보고 detail||message/text 를 통일 파싱해 스타일 modal 로 표시. nudge 만료는 message 문자열이 아니라 code 로 판정.

§3. 변경 상세 (3 항목)

① ErrorModal.fromResponse(resp, opts) — error-modal.js

async function fromResponse(resp, opts) {
    opts = opts || {};
    const ct = resp.headers.get('content-type') || '';
    let message;
    if (ct.includes('json')) {                 // problem+json(미래)·application/json 모두 포함
        const body = await resp.json().catch(() => ({}));
        message = body.detail || body.message || opts.fallback;   // detail = R2-4 미래 무코스트 선반영
    } else {
        message = (await resp.text().catch(() => '')).slice(0, 500) || opts.fallback;
    }
    show({title: opts.title, message: message || '요청을 처리하지 못했어요.', status: resp.status});
}
// window.ErrorModal = {show, hide, fromResponse}

② alert() → ErrorModal 라우팅 (§2 대상만)

  • 응답 객체가 있는 곳: await ErrorModal.fromResponse(resp, {fallback}). 단순 메시지: ErrorModal.show({message}).
  • delete-reject :68 / confirm() / trash-action 결과알림 / showError inline 경로 보존.

③ R2-5 흡수 — nudge 만료 안정 code

  • backend: ApiErrorResponse 에 optional code 필드(@JsonInclude NON_NULL) + ofCode(code,message) factory. ApiExceptionHandler 에 전용 handleNudgeSessionExpired(409 + code="NUDGE_SESSION_EXPIRED") — 특수 body 핸들러라 R2-3 수렴 원칙과 정합(handleNudgeRequired/SoftDeleteReject 와 동류).
  • frontend: nudge-modal.js:89 body.message.indexOf('만료') → body.code === 'NUDGE_SESSION_EXPIRED' (404 NudgeNotFound 분기는 유지).
