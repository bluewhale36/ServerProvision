1.3 footer 의 status label 현재 구현

// background-jobs.js — Line 215~228
function statusLabel(status) {
    switch (status) {
        case 'PENDING':   return '대기';
        case 'RUNNING':   return '진행 중';

2. 요구사항 (확정)

2.1 Notion 명세 (S5-8 본문)

Background Job Panel 에서 status 를 단순히 '진행 중', '완료', '실패' 등으로 표시하던 것에서 어떤 작업을 하고 있는지 간략하게 기술하도록 수정.

2.2 사용자 의도 해석 (자율 결정)

  • title 은 이미 카드 헤더에 크게 노출 — "작업 종류" 는 알 수 있음 (예: "Rocky Linux 9.4 / dvd.iso")
  • type 칩도 헤더 우측에 노출 — JobType 카테고리 ("환경·패키지 추출") 도 알 수 있음
  • 그러나 알림 패널에서 "지금 정확히 어느 단계에 있는가" 는 chunk bar 색상만으로 파악해야 했음 — hover 해야 단계 이름이 보임
  • → 사용자 의도 = 현재 RUNNING 단계 라벨을 footer 에 텍스트로 직접 노출. 시선 1 회로 파악 가능

2.3 4 상태별 텍스트 정책 (확정)

JobStatus현재S5-8 후합성 규칙
PENDING "대기" "대기 중" 고정 문자열 (단계 진입 전, 어떤 단계도 RUNNING 아님)
RUNNING "진행 중" ${runningStage.label} 중 stages 배열 중 status === 'RUNNING' 인 항목의 label. 다중 RUNNING 은 정책상 발생 안 함 (BackgroundJob.startStage 가 이전 RUNNING 을 DONE 으로 전환). 안전 fallback : 없으면 "진행 중"
COMPLETED "완료" "완료" 변경 없음. chunk bar 가 전체 green 이므로 footer 는 깔끔히 "완료" 만
FAILED "실패" ${errorStage.label} 실패 stages 배열 중 status === 'ERROR' 인 항목의 label. fallback : 없으면 "실패"

  • BE 변경 0 — BackgroundJobResponse 의 stages 배열을 그대로 사용. 신규 필드 추가 안 함
  • chunk progress bar 그대로 (S5-8 범위 외)
  • title / subtitle / errorMessage / 닫기 버튼 위치 그대로

3.4 FE 의 currentStatusText 의사 코드

function currentStatusText(job) {
    const status = job.status;
    const stages = Array.isArray(job.stages) ? job.stages : [];

    if (status === 'PENDING')   return '대기 중';
    if (status === 'COMPLETED') return '완료';

    if (status === 'RUNNING') {
        const running = stages.find(s => s.status === 'RUNNING');
        if (running && running.label) return `${running.label} 중`;
        return '진행 중';
    }

    if (status === 'FAILED') {
        const errored = stages.find(s => s.status === 'ERROR');
        if (errored && errored.label) return `${errored.label} 실패`;
        return '실패';
    }

    return status || '';
}

6.3 manual 검증 시나리오 (Step 10 사용자 단독)

  1. RUNNING 단계 진행 — ISO 등록 또는 comps 추출 트리거. 알림 패널 펼쳐 footer 의 "<단계 라벨> 중" 텍스트가 chunk bar 의 RUNNING 청크 (blue pulse) 와 동기 확인. 단계 전환 시 footer 텍스트 즉시 갱신 확인
  2. COMPLETED 전이 — Job 완료 시 footer "완료" 로 단순화 확인. chunk bar 전체 green
  3. FAILED 케이스 — 의도적 실패 유도 (예: ISO 파일 권한 박탈 또는 잘못된 path) → 알림 패널의 footer 가 "<실패한 단계 라벨> 실패" 로 표시되며 chunk bar 의 해당 청크가 red, errorMessage 카드 본문에 별도 노출 확인
  4. 긴 라벨 ellipsis — 등록처 중 라벨이 긴 항목 (예: "manifestHash 재계산") 이 footer 에 들어왔을 때 카드 폭이 깨지지 않고 끝부분이 "..." 으로 잘리는지
  5. PENDING 노출 케이스 — Job 등록 후 startStage 호출 전 (있다면) "대기 중" 으로 표시되는지

응답 이상fallback
status === 'RUNNING' 인데 stages 배열에 RUNNING 항목 0 (정책상 발생 안 함, 동시성 corner case) "진행 중" (기존 동작)
