3. hotfix 2 — 추출 버튼 중복 방지

원인

S5-7 의 RHEL 자동 comps 추출 + S5-12 의 placeholder 가 합쳐지면서, ISO 등록 → background job COMPS_EXTRACTION 자동 트리거 → 사용자가 알림 패널 또는 목록을 보다가 ISO 가 정상 상태로 표시되는 직후 '추출' 버튼이 enabled 상태인 것을 봄 → 또 한 번 클릭 시 같은 자원 중복 추출 시도.

CompsExtractionLauncher 가 동일 isoPath 의 active job 재사용 정책을 가지고 있어 실제 데이터 손상은 발생 안 함. 그러나 UX 측 혼란 + 사용자가 "왜 또 추출되지?" 잘못된 멘탈 모델.

    /** COMPS_EXTRACTION job 이 동일 isoPath 에 대해 active 인 경우 플래그 갱신. */

// OSImageService.assembleIsoResponses 강화

4. hotfix 3 — 4 자원 등록 후 무결성 자동 검증

정책

모든 자원 (BIOS / BMC / Subprogram / OS ISO) 의 등록 메서드가 .provision.json 마커 발급 + manifest 영속 직후 곧장 {...}VerificationLauncher.startVerification() 호출. S5-7 의 RHEL 자동 추출과 같은 사상.

적용 위치 (총 7 호출 추가)

Service메서드호출 헬퍼
BiosServiceaddBiostriggerAutoVerification(boardId, biosId)
BiosServiceregisterExisting동일
BiosServicepersistFromNudge동일
BmcServiceaddBmctriggerAutoVerification(boardId, bmcId)
BmcServiceregisterExisting동일
BmcServicepersistFromNudge동일
SubprogramServiceaddSubprogramtriggerAutoVerification(subprogramId)
SubprogramServiceregisterExisting동일
SubprogramServicepersistFromNudge동일
OSImageServicefinalizePreparedIsoRegistrationtriggerAutoVerification(osImageId, isoId)

fail-safe

private void triggerAutoVerification(Long ...) {
    try {
        {...}VerificationLauncher.startVerification(...);
    } catch (RuntimeException e) {
        log.error("[XXX hotfix] 자동 무결성 검증 트리거 실패 (등록은 영속됨). ...", e);
    }
}

verification launcher 실패는 등록 트랜잭션과 무관 — 등록은 영속됐고, 검증 자체가 실패하면 BackgroundJob 의 FAILED 상태로 알림 패널에 노출. 사용자가 인지하고 수동 재검증 가능.

의존성 추가

  • BiosService → BiosVerificationLauncher
  • BmcService → BmcVerificationLauncher
  • SubprogramService → SubprogramVerificationLauncher
  • OSImageService → IsoVerificationLauncher

각 launcher 는 이미 존재 (사용자 수동 검증 흐름에서 사용). 본 hotfix 는 등록 직후 자동 호출만 추가.

사용자 가시 변경

  • BIOS / BMC / Subprogram / OS ISO 등록 → 알림 패널에 자동으로 INTEGRITY_VERIFICATION job 카드 등장
  • "서명 검증 중 → 해시 재계산 중 → 완료" 진행 표시 (S5-8 의 chunk + 단계 라벨)
  • 완료 후 자원의 무결성 status (ORIGINAL / SIGNATURE_INVALID / TAMPERED) 가 즉시 UI 에 반영

RHEL ISO 의 경우 동시 트리거되는 3 job

  1. ISO_REGISTRATION (S1 / 본래) — 해시 계산 / 중복 검사 / 메타 저장
  2. COMPS_EXTRACTION (S5-7) — RHEL family 자동 comps 추출
  3. INTEGRITY_VERIFICATION (본 hotfix) — .provision.json 서명 검증 + manifestHash 재계산

모두 병렬 진행 — 각 job 의 첫 stage 가 파일 존재 / 읽기 가능성 확인하므로 안전. 사용자는 알림 패널에서 3 job 동시 진행 관찰.
