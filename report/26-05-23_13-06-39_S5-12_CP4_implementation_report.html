1. 요약

OS ISO 등록 시 background job (해시 계산 / 중복 검사 / DB 영속) 진행 중 OS 상세 패널의 ISO 목록에 "등록 중" placeholder를 표시. 사용자가 등록한 자원이 시스템에 들어왔음을 즉시 시각 확인.

  • 출처 : BackgroundJobService.snapshot() 의 활성 ISO_REGISTRATION job (metadata.osId 매칭, subtitle = isoPath)
  • 합성 시점 : OSImageService 의 ISO 목록 조립 시점 (findById / findAllGrouped 양쪽)
  • 중복 회피 : DB 영속 직후 race 케이스 — 이미 같은 isoPath 의 정상 ISO 가 base 에 있으면 placeholder 미추가
  • UX : opacity 0.65 + 회색 배경 + "등록 중" 파란 배지 + 액션 부재 + 안내 문구
  • 자동 reload : background-jobs.js 의 bgjob:completed 이벤트 활용 — type=ISO_REGISTRATION 완료 시 페이지 reload
채택 = 안 A (in-memory BackgroundJob 합성) — DB 변경 0. BackgroundJob 의 in-memory 수명 cycle 과 placeholder 도 같은 lifecycle. JVM 재시작 시 진행 중이던 job 자체도 사라지므로 일관.

2.3 OSImageService — assembleIsoResponses 헬퍼

private List<ISOResponse> assembleIsoResponses(OSImage image, boolean includeDeleted) {
    List<ISOResponse> base = visibleISOs(image.getIsos(), includeDeleted).stream()
            .map(ISOResponse::from)
            .collect(Collectors.toCollection(ArrayList::new));

    Set<String> existingPaths = base.stream()
            .map(ISOResponse::isoPath)
            .collect(Collectors.toSet());

    String osIdString = String.valueOf(image.getId());
    backgroundJobService.snapshot().stream()
            .filter(j -> j.getType() == JobType.ISO_REGISTRATION)
            .filter(j -> j.getStatus().isActive())
            .filter(j -> osIdString.equals(j.getMetadata().get("osId")))
            .filter(j -> !existingPaths.contains(j.getSubtitle()))
            .forEach(j -> base.add(ISOResponse.inProgress(j.getSubtitle())));

    return base;
}

findById / findAllGrouped 의 OSImageResponse.of(...) 호출 → ofAssembled(image, assembleIsoResponses(image, ...), ...) 로 갱신.

3. 동작 흐름
  1. 사용자 POST /management/os/{osId}/iso 호출 (파일 업로드)
  2. OSImageService.prepareIsoRegistration → 파일 저장 + sidecar 마커 (foreground)
  3. IsoRegistrationLauncher.startRegistration() → BackgroundJob ISO_REGISTRATION 등록 (metadata.osId / subtitle=isoPath)
  4. 응답 즉시 반환 → OS 목록 페이지로 redirect
  5. 사용자가 OS 상세 패널 → OSImageService.findById() 호출 → assembleIsoResponses 호출 → 진행 중 job 합성
  6. list.html 렌더 → placeholder ISO 가 "등록 중" 배지 + 회색 + 안내 문구로 표시
  7. background-jobs.js 가 PERSIST_METADATA 완료 시 bgjob:completed 이벤트 dispatch → 페이지 reload → placeholder 가 정상 ISO 로 교체

5. 사용자 가시 변경
  • Before : ISO 등록 폼 제출 → OS 목록 redirect → 새 ISO 가 목록에 안 보임 → 사용자 "등록 안 됐나?" 혼란 → 알림 패널 펼쳐서 확인
  • After : ISO 등록 폼 제출 → OS 목록 redirect → 새 ISO 가 "등록 중" placeholder 로 즉시 표시 → 알림 패널 + 목록 양쪽에서 진행 상황 확인 → 완료 시 자동 reload → 정상 ISO 로 교체

5.1 다중 동시 등록

한 OS 에 여러 ISO 동시 등록 (예: dvd.iso + minimal.iso) → 모두 placeholder 표시. 정렬은 별도 처리 안 함 (BackgroundJobService.snapshot() 의 시간 역순).

5.2 실패 시

BackgroundJob 이 FAILED 로 전환되면 isActive() false → placeholder 자동 제거. 사용자는 알림 패널의 FAILED 카드에서 실패 원인 확인 가능 (S5-8 의 errorStage 라벨 + errorMessage).

5.3 다른 사용자 / 탭

BackgroundJob 이 JVM 메모리에 있고 동일 GET 요청에 같은 합성 결과 반환되므로, 다른 브라우저 / 탭 / 사용자가 같은 OS 조회해도 동일하게 placeholder 보임. 운영 일관성.
