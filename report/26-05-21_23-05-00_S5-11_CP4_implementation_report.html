1. 요약

BIOS 번들 진입점 자동 탐지 우선순위에 ASUS .cap / .CAP 단계를 신설. 기존 단계 (트리 전체 단일 파일 / f.nsh / flash.nsh / 단일 .nsh) 의 fall-through 사이에 삽입.

  • 이전 한계 : ASUS .CAP + 부속 README / 도구 동봉 시 단계 2 (단일 파일) fail → 단계 3-5 (.nsh) 가 .cap 미인식 → EntrypointNotFoundException
  • S5-11 후 : 루트 수준 단일 .cap 자동 채택. ASUS 의도 진입점 정확 식별
  • 대소문자 무시 : .cap / .CAP / .Cap 모두 동일 취급
  • 충돌 정책 : .cap + f.nsh 동시 존재 시 .cap 우선 (ASUS 가 의도된 진입점)
Fujitsu 제외 Notion 명시. iRMC 흐름은 별 슬라이스. 본 슬라이스는 ASUS + 기존 GIGABYTE 회귀 보장.
2. 변경 내용

2.1 BundleEntrypointDetector.detect()

// 2) 트리 전체 파일 1 개면 그것 — 기존 그대로
List<Path> allFiles = listRegularFilesExcludingMarker(treeRoot);
if (allFiles.size() == 1) return toRelative(treeRoot, allFiles.get(0));
if (allFiles.isEmpty()) throw new EntrypointNotFoundException("번들에 진입점으로 쓸 수 있는 파일이 없습니다.");

// 3) S5-11 — 제조사별 정합화 : 루트 수준 *.cap / *.CAP (ASUS 컨벤션) 가 정확히 1 개면 그것 채택.
//    .CAP + 부속 README / .bin 등이 동봉된 ASUS 번들에서 단계 2 (전체 트리 단일 파일) 가 실패하던 케이스 해소.
//    2 개 이상이면 ambiguous, 0 개면 .nsh 단계로 fall-through.
List<Path> rootCap = listRootCapFiles(treeRoot);
if (rootCap.size() == 1) return toRelative(treeRoot, rootCap.get(0));

// 4~6) 루트 .nsh — 기존 GIGABYTE 컨벤션
List<Path> rootNsh = listRootNshFiles(treeRoot);
if (fNsh != null) return "f.nsh";
if (flashNsh != null) return "flash.nsh";
if (rootNsh.size() == 1) return toRelative(treeRoot, rootNsh.get(0));

// 다중 .cap 또는 다중 .nsh → ambiguous (후보 리스트에 .cap + .nsh 합집합 포함)
if (rootCap.size() >= 2 || rootNsh.size() >= 2) {
    List<String> candidates = Stream.concat(rootCap.stream(), rootNsh.stream())
            .map(p -> toRelative(treeRoot, p)).sorted().distinct().toList();
    throw new EntrypointAmbiguousException(candidates);
}

throw new EntrypointNotFoundException(
    "트리 루트에서 *.cap (ASUS) · f.nsh · flash.nsh · 단일 *.nsh 어느 것도 발견되지 않았습니다. 진입점을 명시 지정하세요.");

2.2 신규 헬퍼 — listRootCapFiles

/**
 * S5-11 — 루트 수준의 *.cap / *.CAP 후보 수집 (ASUS BIOS 컨벤션). 대소문자 무시.
 * 마커 / OS 잡파일 (.DS_Store / Thumbs.db) 자동 제외.
 */
private List<Path> listRootCapFiles(Path treeRoot) {
    List<Path> result = new ArrayList<>();
    try (Stream<Path> children = Files.list(treeRoot)) {
        children.filter(Files::isRegularFile)
                .filter(p -> p.getFileName().toString().toLowerCase().endsWith(".cap"))
                .filter(p -> !BundleFilePolicy.isIgnorable(p))
                .forEach(result::add);
    } catch (IOException e) {
        throw new BundleExtractionException("루트 .cap 스캔 실패 : " + treeRoot, e);
    }
    return result;
}
