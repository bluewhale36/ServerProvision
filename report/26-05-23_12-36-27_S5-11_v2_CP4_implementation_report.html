3. Strategy 본문

3.1 ASUS AsusEntrypointStrategy

우선순위 : override → 트리 단일 파일 → 루트 *.cap 1 개 → fallback 단일 *.nsh → ambiguous / notFound

// 3) 루트 *.cap 1 개 → 그것 (.CAP + 부속 케이스)
List<Path> rootCap = listRootFilesByExtension(treeRoot, ".cap");
if (rootCap.size() == 1) return toRelative(treeRoot, rootCap.get(0));

// 4) fallback : .cap 0 + 단일 .nsh → 그것 (관용)
List<Path> rootNsh = listRootFilesByExtension(treeRoot, ".nsh");
if (rootCap.isEmpty() && rootNsh.size() == 1) return toRelative(treeRoot, rootNsh.get(0));

// 5) ambiguous / notFound

3.2 GIGABYTE GigabyteEntrypointStrategy

우선순위 : override → 트리 단일 파일 → f.nsh → flash.nsh → 단일 *.nsh → ambiguous / notFound. .cap 동봉 자동 채택 안 함

List<Path> rootNsh = listRootFilesByExtension(treeRoot, ".nsh");
Path fNsh = findByName(rootNsh, "f.nsh");
Path flashNsh = findByName(rootNsh, "flash.nsh");

if (fNsh != null) return "f.nsh";
if (flashNsh != null) return "flash.nsh";
if (rootNsh.size() == 1) return toRelative(treeRoot, rootNsh.get(0));

// .cap 동봉되어 있어도 GIGABYTE 정책상 자동 채택 안 함 — 메시지에 안내만
if (rootNsh.size() >= 2) throw ambiguous(...);
throw notFound("...*.cap 은 GIGABYTE 정책상 자동 채택되지 않으니 필요 시 override 로 명시...");

4. Dispatcher 
@Service
@RequiredArgsConstructor
public class BundleEntrypointDetector {
    private final List<EntrypointDetectionStrategy> strategies;

    public String detect(Vendor vendor, Path treeRoot, String override) {
        return strategies.stream()
                .filter(s -> s.supports(vendor))
                .findFirst()
                .orElseThrow(() -> new EntrypointStrategyMissingException(vendor))
                .detect(treeRoot, override);
    }
}

구버전의 모든 vendor 통합 정책 / private helper 들 → 제거. 본문은 strategy 위임 1 줄. EntrypointStrategyMissingException 으로 새 vendor 추가 시 구현체 누락을 즉시 드러냄.
