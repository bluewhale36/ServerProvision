1. 요약

4 도메인 (BIOS / BMC / OS / Subprogram) 의 디렉토리 탐색 패널에서 dot-prefix 파일 / 폴더 또는 OS-level hidden 속성 가진 entry 를 회색 (muted color + opacity 0.55) 으로 표시. ISO highlight (파란색) 같은 다른 시각 처리와 충돌 시 hidden 우선.

  • BE : DirectoryListingResponse.Entry 에 hidden: boolean 필드 추가. name.startsWith('.') || Files.isHidden(p) 통합 판별
  • FE : path-browser.js renderEntries 에서 e.hidden 시 inline style muted + opacity. hover / click 동작 유지
  • Factory 호환 : 기존 Entry.directory(name) / Entry.file(name, size) 시그니처 보존 (hidden=false default) + 신규 4-arg factory 추가
S5-10 연계 S5-10 의 Files.setAttribute("dos:hidden", true) 부여 효과를 본 슬라이스의 BE 판별이 자연 흡수. Windows 환경에서도 회색 표시.
2. 변경 내용

2.1 DirectoryListingResponse.Entry — record 1 필드 + factory 2 신규

public record Entry(
    String type,
    String name,
    Long size,
    boolean hidden
) {
    // 기존 호출자 호환 (테스트 3 곳) — hidden=false default
    public static Entry directory(String name) { return new Entry("DIR", name, null, false); }
    public static Entry file(String name, long size) { return new Entry("FILE", name, size, false); }

    // S5-9 신규 — service 측에서 hidden 판별 결과 전달
    public static Entry directory(String name, boolean hidden) { return new Entry("DIR", name, null, hidden); }
    public static Entry file(String name, long size, boolean hidden) { return new Entry("FILE", name, size, hidden); }
}

2.2 DirectoryBrowseService — hidden 계산 추가

for (Path p : sorted) {
    if (entries.size() >= maxEntries) { truncated = true; break; }
    String name = p.getFileName().toString();
    // S5-9 — dot-file 관례 OR OS-level hidden (Windows/macOS) 통합 판별
    boolean hidden = name.startsWith(".") || isOsHidden(p);
    if (Files.isDirectory(p)) {
        entries.add(DirectoryListingResponse.Entry.directory(name, hidden));
    } else if (request.includeFiles()) {
        long size = -1L;
        try { size = Files.size(p); } catch (IOException ignore) {}
        entries.add(DirectoryListingResponse.Entry.file(name, size, hidden));
    }
}

private boolean isOsHidden(Path p) {
    try { return Files.isHidden(p); }
    catch (IOException e) { return false; }
}

4. 사용자 가시 변경
  • BIOS / BMC / OS / Subprogram 등록 폼의 탐색 패널에서 .provision.json / .cache / .git 등이 회색 + opacity 0.55 로 dim
  • ISO 파일 (highlight 파란색) 은 일반 entry 와 시각 분리 그대로 유지

의도 사용자가 등록할 자원 (예: ISO / BIOS 번들) 을 한 눈에 식별 가능. 시스템 / 메타 파일을 자원으로 오인 선택하는 risk 차단.
