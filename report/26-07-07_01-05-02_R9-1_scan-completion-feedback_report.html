사용자 불만 ① "스캔 후 화면을 새로고침 할 필요가 있다"를 해소했다. 이 페이지에서 트리거한 스캔이 완료되면 완료 토스트("점검 완료 — 드리프트 N건")와 함께 자동 새로고침되고, 사용자가 트리거하지 않은 완료(1시간 주기 스캔 등)는 열람 중 화면이 리셋되지 않도록 배너("새 점검 보고서가 도착했습니다 — [새로고침]")로만 안내한다. 마커 서명 재발급은 보고서를 만들지 않으므로 새로고침 없이 성공/실패 건수 토스트만 표시 — 기존에 서버 로그로만 새던 부분 실패(불만 ③의 일부)가 처음으로 화면에 도달한다.

구현의 골격은 신규 발명이 아니라 접속이다: 완료 감지는 이미 전 페이지에서 돌던 알림 센터 폴링(background-jobs.js)의 bgjob:completed/failed 이벤트를 구독했고, 완료 시점 결과 수치는 BackgroundJobService.complete(jobId, 결과 map) 오버로드가 등록 metadata 와 병합해 기존 wire 계약(metadata 필드) 그대로 내려보낸다 — 전역 JS 는 한 줄도 바뀌지 않았다.

부수 결함 2건 동반 수정 — ⓐ 재발급 Job 이 스캔용 3단계 라벨("디스크 스캔/드리프트 분류/보고서 저장")을 차용해 알림 센터 진행바가 거짓 단계를 표시하던 것을 전용 ReissueStage("마커 재서명" 1단계)로 교체. ⓑ Miller 드릴다운 후 새로고침하면 선택 위치가 항상 초기화되던 것(os-list.js 의 URL 동기화 파라미터 selectKey/selectId 를 컨트롤러가 안 읽음)을 alias 수용으로 수정 — 자동 reload UX 의 전제 결함이었다.

§6. CP5 가이드 — 사용자 브라우저 E2E
  1. 수동 스캔 — 자원 경로 점검 페이지에서 [수동 스캔] 클릭 → 시작 토스트 → 버튼 "스캔 중…" 유지 → 완료 시 자동 새로고침 + "점검 완료 — 드리프트 N건" 토스트. 알림 센터(서류가방) 카드의 진행바가 3단계(디스크 스캔→드리프트 분류→보고서 저장)로 채워지는지 확인.
  2. 위치 보존 — 스캔 이력에서 보고서·드리프트를 드릴다운한 상태로 F5 → 선택 위치가 유지되는지 (URL 의 selectKey/selectId).
  3. 마커 서명 재발급 — 클릭(confirm 통과) → 완료 시 새로고침 없이 "성공 N건, 실패 M건" 토스트. 알림 센터 카드 진행바가 "마커 재서명" 1단계인지.
  4. 주기 스캔 배너(선택) — 페이지를 열어둔 채 다른 브라우저 탭/세션에서 스캔 트리거 → 열람 중 화면이 리셋되지 않고 상단 배너 + [새로고침] 버튼이 뜨는지.
