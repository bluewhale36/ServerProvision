§1. 구현 요약

S6-2 캠페인의 기반 슬라이스 — 신규 kind 를 아직 추가하지 않고, 신규 kind 가 안전하게 꽂힐 세 가지 구조를 완성했다:

  • 해결 디스패치 다형화 — apply()/스캔 무인 적용의 kind if-else 를 DriftResolution 전략 bean Map 디스패치로 치환. 기존 2종(PATH_DRIFT/GHOST_DB_ROW)은 동작 등가 이전. Path.of(null) NPE 잠복 경로 소멸(테스트로 고정).
  • DriftResolutionMode 3단(NONE/MANUAL/AUTO) — boolean 1축을 대체. MANUAL(사용자 확인 후 시스템이 해결) 개념이 타입으로 생겼고, 불법 조합(manual=false·auto=true)은 표현 자체가 불가.
  • 설정 통합·개명(breaking 일괄 1회) — kind 별 boolean 2키 → reconciliation.auto-apply.kinds CSV 1키, 마스터 auto-apply → resolution-enabled. 예외도 DriftResolutionNotAllowedException 으로 개명해 설정·가드·문구·UI tooltip 이 전부 "시스템 해결" 어휘로 정합.

§5. 동작·설정 변화 (CP5 안내)
설정 breaking (실효 영향 0, 파일 정리 권장) — reconciliation.auto-apply-path-drift/auto-apply-ghost-row/reconciliation.auto-apply 세 키는 더 이상 읽히지 않는다(남아 있어도 조용히 무시). 대체: reconciliation.resolution-enabled=true(마스터) + reconciliation.auto-apply.kinds=(무인 자동 허용 kind CSV, 빈 default = 전부 수동). 저장소 application.properties 는 정비 완료 — 로컬/운영 override 파일에 구 키가 있으면 정리 필요.
  • 화면 변화: [자동 적용] 버튼 → [적용], 마스터 OFF tooltip "시스템 해결 옵션이 꺼져 있어요 (reconciliation.resolution-enabled)". C3 배지가 3색(주황=자동 가능/노랑=확인 후 해결/회색=안내만) — 현재 노랑은 등장할 kind 가 없어 S6-2-2 부터 실표시.
  • 409 거절 문구가 "자동 정합" → "시스템이 해결할 수 없습니다" 계열로 갱신(direct POST 안전망에서만 노출).
