1. 요약

알림 패널 카드 footer 의 status 텍스트를 현재 진행 중인 단계 라벨 기반으로 합성하도록 변경. 사용자가 "지금 무엇을 하고 있는가" 를 시선 1 회로 파악 가능.

  • RUNNING : ${runningStage.label} 중 (예: "comps.xml 파싱 중", "manifestHash 재계산 중")
  • FAILED : ${errorStage.label} 실패 (예: "서명 검증 실패")
  • PENDING / COMPLETED : "대기 중" / "완료" (단순화)

안 A 채택 — FE-only 변경. BE 응답 구조 0 변경. BackgroundJobResponse.stages[i] 의 (label, status) 페어를 그대로 사용.

핵심 가치 chunk progress bar 색상 = "어디까지 왔는가", footer 라벨 = "지금 정확히 무엇을 하는가". 두 시각 요소가 보완적으로 작동.
2. 변경 내용

2.1 background-jobs.js

기존 statusLabel(status) 함수를 currentStatusText(job) 로 시그니처 확장. job 객체 전체를 받아 stages 배열에서 RUNNING / ERROR 단계 추출.

function statusLabel(status) {
    switch (status) {
        case 'PENDING':   return '대기';
        case 'RUNNING':   return '진행 중';
        case 'COMPLETED': return '완료';
        case 'FAILED':    return '실패';
        default: return status || '';
    }
}
function currentStatusText(job) {
    if (!job) return '';
    const status = job.status;
    const stages = Array.isArray(job.stages) ? job.stages : [];

    if (status === 'PENDING') return '대기 중';
    if (status === 'COMPLETED') return '완료';

    if (status === 'RUNNING') {
        const running = stages.find(s => s && s.status === 'RUNNING');
        if (running && running.label) return `${running.label} 중`;
        return '진행 중';
    }

    if (status === 'FAILED') {
        const errored = stages.find(s => s && s.status === 'ERROR');
        if (errored && errored.label) return `${errored.label} 실패`;
        return '실패';
    }

    return status || '';
}

cardHtml() 호출부 갱신 — XSS 방어 추가 (escapeHtml):

<span class="n-bgjob-card-status">${statusLabel(job.status)}${escapeHtml(currentStatusText(job))}</span>

5. 사용자 가시 변경

5.1 카드 footer 변화

  • Before : "진행 중" (모든 RUNNING Job 동일) / "실패" (모든 FAILED Job 동일)
  • After : "comps.xml 파싱 중" / "manifestHash 재계산 중" / "해시 재계산 실패" 등 단계별 명확한 표현

5.2 chunk progress bar 와의 보완

  • chunk 색상은 "단계의 진행 위치" 시각화 (DONE 청크 + RUNNING 청크의 pulse)
  • footer 라벨은 "현재 단계의 이름" 텍스트 노출
  • 두 시각 요소가 상호 보완 — 사용자가 hover 없이도 단계 식별 가능

5.3 ellipsis 처리

긴 라벨도 카드 폭 깨지지 않고 "..." 으로 자연 잘림. max-width: 60% 로 시각 60% 점유 + sibling time 의 우측 정렬 보존.
